# Docs

This folder contains reusable research references and templates.

Use root-level markdown files for global project workflow.
Use `docs/` for detailed reference material that is useful but not necessary
for every minor code change.

## Files

- `EXPERIMENT_TEMPLATE.md`: standard structure for experiments
- `EVALUATION.md`: posterior evaluation principles
- `CODING_STANDARDS.md`: coding and reproducibility standards
- `SBI_NOTES.md`: compact SBI reference notes
- `FLOW_MATCHING_NOTES.md`: compact flow-matching reference notes

# 3.1 Evaluation

## Purpose

The current experiment evaluates whether BO-guided simulation design improves posterior learning compared with random design.

The experiment compares two design policies:

- **random:** design variables $\psi$ are sampled randomly,
- **bo:** design variables $\psi$ are selected by Bayesian optimization.

In both cases, physical parameters are still sampled from the prior:

$$
\theta \sim p(\theta).
$$

BO only controls the simulation or experiment design:

$$
\psi = \text{design variables}.
$$

For the Lotka-Volterra prototype, the hard-window design uses:

$$
\psi = (t_{\text{span}}, t_{\text{start}}),
$$

with fixed initial populations. The full design additionally includes initial prey and predator populations.

## Posterior Quality

This panel shows:

$$
J_{\text{LP}}[q_\phi]
=
\frac{1}{|\mathcal{V}|}
\sum_{(\theta_i,\psi_i,x_i)\in\mathcal{V}}
\log q_\phi(\theta_i\mid x_i,\psi_i).
$$

Here $\mathcal{V}$ is a held-out validation set of simulator-generated examples.

Each validation example has a known simulator parameter:

$$
\theta_i \sim p(\theta),
\qquad
x_i \sim p_{\text{sim}}(x\mid\theta_i,\psi_i).
$$

The score asks:

> Does the learned posterior assign high probability density to the true parameter that generated each validation observation?

Higher is better.

This does **not** reward the model for moving toward one predefined parameter. Every validation point has its own ground-truth $\theta_i$.

In the implementation, this is the column:

```text
validation_log_posterior
```

## Calibration Proxy

This panel shows a simple marginal coverage error.

For each validation example, the Gaussian posterior baseline predicts a posterior mean and a residual covariance. For several nominal credible levels,

$$
\alpha \in \{0.5, 0.8, 0.9\},
$$

we check whether the true standardized parameter lies inside the corresponding marginal Gaussian credible interval.

The error is:

$$
E_{\text{cov}}
=
\frac{1}{|A|}
\sum_{\alpha\in A}
\left|
\widehat{\operatorname{coverage}}(\alpha)
-
\alpha
\right|.
$$

Lower is better.

Interpretation:

> If the model says “this is an 80% credible interval,” then the true parameter should fall inside about 80% of the time.

This is only a proxy because the current baseline is a simple Gaussian posterior model. Later, with FMPE, this should be replaced or supplemented by stronger SBI diagnostics such as SBC, expected coverage, TARP, or local coverage tests.

In the implementation, this is the column:

```text
coverage_error
```

## BO Reward Trace

This panel shows the scalar reward that BO receives after selecting a design.

The current reward is improvement in validation log posterior:

$$
u_t(\psi_t)
=
J_{\text{LP}}[q_{\phi_{t+1}}]
-
J_{\text{LP}}[q_{\phi_t}].
$$

Here:

- $q_{\phi_t}$ is the posterior estimator before adding the BO-selected simulations,
- $q_{\phi_{t+1}}$ is the posterior estimator after adding those simulations and retraining,
- $\psi_t$ is the design selected by BO.

Positive reward means:

> The selected design improved validation posterior quality after updating the model.

Negative reward means:

> The selected design made validation posterior quality worse, or the training noise dominated the true benefit.

In the implementation, this is the column:

```text
reward_delta_log_posterior
```

This is the noisiest plot because each reward depends on simulator randomness, posterior-model training, and validation evaluation.

## BO-Selected Designs

This panel shows which physical design values BO selected over rounds.

For the Lotka-Volterra simulator, design variables can include:

- `prey0`: initial prey population,
- `pred0`: initial predator population,
- `t_start`: start time of the observation window,
- `t_span`: length of the observation window.

In `hard_window` mode, only the observation window is optimized:

$$
\psi = (t_{\text{span}}, t_{\text{start}}),
$$

while:

$$
\text{prey0}=40,
\qquad
\text{pred0}=15.
$$

This plot asks:

> Is BO repeatedly selecting certain observation windows or experimental conditions?

If BO works well, one might expect it to discover designs that are more informative, for example observation windows that capture predator-prey oscillations rather than flat or uninformative parts of the trajectory.

## How To Interpret The Four Panels Together

The desired pattern is:

1. **Posterior Quality:** BO rises faster than random.
2. **Calibration Proxy:** BO has equal or lower coverage error.
3. **BO Reward Trace:** rewards become more often positive or less volatile.
4. **BO-Selected Designs:** BO converges toward interpretable informative designs.

If BO improves posterior quality but worsens calibration, the posterior may become overconfident.

If BO reward is unstable, the scalar reward is probably too noisy for BO.

If BO-selected designs jump around without pattern, the acquisition function may not be learning a useful design landscape.

If random design performs as well as BO, then either:

- most designs are already informative,
- the design variables are not important enough,
- the reward is too noisy,
- the posterior model cannot exploit the selected simulations,
- or the validation set does not match the design objective.

## Current Status

The current experiment should be treated as a diagnostic prototype, not evidence for the final claim.

The important question is not only:

$$
\text{Does BO beat random?}
$$

but also:

$$
\text{Can we define a stable scalar posterior-quality reward for BO?}
$$

That reward-design problem is the central methodological issue of the project.
